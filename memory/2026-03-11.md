# 2026-03-11

- CPA/CLIProxyAPI 通道模型体检：对 `cpa/gpt-5.4`、`gpt-5.3-codex`、`gpt-5.3-codex-spark`、`gpt-5-codex`、`gpt-5-codex-mini`、`gpt-5.2-codex`、`gpt-5.2` 逐个冒烟，结果 0/7 全失败。
- 现象：CLIProxyAPI `GET /v1/models` 正常返回模型列表，但 `POST /v1/chat/completions` 全部在 25s/40s/65s 后报 `context canceled`，重启 `cliproxyapi.service` 后仍复现。
- 根因确认：抽样直连 OpenAI 官方接口校验多个 `~/.cli-proxy-api/*.json` 账号的 `access_token`，均返回 `401 token_invalidated`。说明不是候选数不够，而是当前 CPA 账号池中的 token 整体已失效。
- 管理接口中的 `candidates=200` 仅表示文件/候选条目存在，不代表 token 仍可实际调用。
- 后续建议：执行 `cpaclean`/`auto_pool_maintainer.py` 做一次正式维护（清理失效账号 + 补号 + 收敛），完成后再复测全模型。
